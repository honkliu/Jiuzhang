# 用户帮助

## 聊天

在 **聊天** 中可以和个人或群组沟通，支持文字、图片、语音，也可以让 AI 辅助生成内容。

聊天命令:

- `/h` 显示聊天命令帮助。
- `/w` 查看聊天成员。
- `/wa` 查看当前活跃成员。
- `/b <文字>` 发送粗体。
- `/i <文字>` 发送斜体。
- `/r <文字>` 发送红色文字。
- `/p [提示词]` 生成图片；群聊里可用 `/p @某人 [提示词]` 或 `/p @某人 @某人 [提示词]` 指定人物。
- `/a @某人` 把某人加入当前聊天。
- `@名字` 提及某个人。
- `@@` 呼叫洛。

示例:

- `@@ 帮我总结这个计划`
- `/p @Amy @Ben 穿正式婚礼服装` 会按提及的人生成图片。

2D 沉浸式聊天用法:

- 打开一个一对一聊天。
- 点击聊天标题栏右侧的 2D 按钮进入沉浸式视图。
- 左右两边会分别显示对方和自己的最近消息，更适合轻松对话。
- 再点一次 2D 按钮可退出，回到普通聊天列表。

## bbtalk

bbtalk 是命令行聊天工具，适合在终端里快速聊天、看长回答、看 Markdown 或公式输出。

bbtalk 基本用法:

- 第一次使用先进入 `KanKan/bbtalk`，执行 `npm install`。
- 登录: `node index.js login <email> <password> --base-url <api-base-url>`。
- 启动: `npm start` 或 `node index.js`。
- 输入消息后按 Enter 发送；按 `Ctrl+C` 退出。

bbtalk 聊天命令:

- `/cl` 列出聊天。
- `/cj <编号>` 进入 `/cl` 列表中的聊天。
- `/cq` 离开当前聊天。
- `/cd` 清空/删除当前聊天记录。
- `/cn` 显示当前聊天名称。

bbtalk 联系人命令:

- `/ul` 列出用户。
- `/ua <编号>` 向 `/ul` 列表中的用户发送好友请求。
- `/c <编号>` 和 `/ul` 列表中的用户创建一对一聊天。
- `/ur` 查看待处理好友请求。
- `/urc [编号]` 接受某个请求；不写编号时接受全部。
- `/urd [编号]` 拒绝某个请求；不写编号时拒绝全部。

bbtalk 其他命令:

- `/du <编号>` 删除 `/ul` 列表中的用户；仅管理员可用。
- `/help` 显示帮助。
- `/quit` 退出程序。

## 图片美化

点击图片进入灯箱后，可以缩放、浏览原图、查看已生成结果，也可以输入提示词生成新的美化图。

基本流程:

- 从聊天、啪、票夹、图览或家谱页面打开图片。
- 输入想要的效果，然后点击美化。
- 用提示词库按钮选择现成效果。
- 用调节按钮快速选择风格或表情。
- 需要两张图合成时，选择另一张图作参考。

示例:

- `把背景换成安静的中式庭院`
- `把这张头像变成正式证件照风格`
- `去掉背景里的人`

## 札记

**札记** 适合写长内容：会议记录、草稿、家族资料、学习笔记，或者任何需要整理的内容。

使用要点:

- 用笔记本下拉框切换不同札记本。
- 顶部彩色标签是章节。
- 用页码和左右箭头切换页面。
- 双击章节名可以改名。
- 用导入/导出做备份或迁移。

示例:

- 创建 `项目A`，再加 `想法`、`会议`、`决策` 三个章节。

## 家谱

**家谱** 用来查看人物、关系、辈分和家族谱志页面。点击人物可查看详情、照片、备注、配偶、父母和子女。

常用操作:

- 在人物面板里添加父母、配偶和子女。
- 简注适合短标签，不适合长故事。
- 长故事放到家族谱志里更合适。
- 用关联人物连接不同家谱里的同一个人。

出继与承继:

- `出继` 表示子女从本支出继到另一支承继。
- 系统会在承继人名下生成对应的 `继子` 关系。
- 保存前要仔细选择承继的伯叔或分支。

示例:

- 把 `邵氏主谱` 中的人关联到 `邵氏支谱` 的同名人物。
- 次子承继伯叔一支时，可标记为 `出继`。

## 票夹

**票夹** 用来保存票据照片和识别结果。上传照片后检查识别内容，需要时再修正字段。

小技巧:

- 出行或就诊后适合批量上传。
- 图片集合适合按视觉方式浏览票据照片。
- OCR 是草稿，金额、日期、医院名最好检查一下。

示例:

- 上传几张医疗票据，先修正日期和金额，之后再检索。

## 啪

**啪** 是轻量分享空间。想快速发状态或图片，但不需要正式聊天或札记时，用它就好。

示例:

- 发一张旅行照片，加一句简短说明。
